# GameCP

## 项目介绍

本仓库用于存放北师大数学建模复赛参赛作品—— **《基于AHP-熵权法-TOPSIS 组合模型的游戏性价比综合评价模型》** 的相关文件代码及数据。

## 文件结构

- `Group Game`文件夹
  - 此文件夹存放了一部分的游戏分组记录及相关数据。
  - `log` 文件夹下存放了运算时相关数据，命名规则为 `validation_result_分组编号.txt`。
  - `rawData` 文件夹下存放了游戏分组的原始数据，命名为`各组名.csv`。
  - `result` 文件夹下存放了游戏分组的运算结果，命名为`result_分组编号.txt`。
  - 分组编号:
       | 编号       | 分组 |
       | --- | ---------- |
       | 1   | 策略类     |
       | 2   | 动作类     |
       | 3   | 角色扮演类 |
       | 4   | 解谜类     |
       | 5   | 竞速类     |
       | 6   | 冒险类     |
       | 7   | 模拟类     |
       | 8   | 射击类     |
       | 9   | 体育类     |
       | 10  | 休闲类     |
- `Questionnaire`文件夹
  - 此文件夹存放了问卷相关数据。
- ` Total Game `文件夹
  - 此文件夹存放了所有游戏合在一起的记录及相关数据。
- `Code`文件夹
  - 此文件夹存放了项目的代码文件。
  - `游戏爬取工具.exe` 是一个用于爬取游戏数据的工具，用于获取游戏的详细信息。
    - 此工具将爬取 **Steam平台** 上指定 **ID** 的游戏详细信息，包括游戏名称、游戏类型、游戏评分、游戏价格等。同时将爬取 **小黑盒** 平台上指定 **ID** 的游戏信息，包括游戏平均游玩时长。需要注意的是，**小黑盒** 平台上的游戏信息是基于 **HowLongToBeat** 平台获取，我们一开始也是想直接从 **HowLongToBeat** 平台获取游戏信息，但是由于测试时无法正确访问其API，故选择从 **小黑盒** 爬取平均游玩时长。
    - 特别的，**小黑盒** 的游戏id与 **Steam平台** 的游戏id相同，已保存在同目录下`gameid.json`文件中。此文件使用 **Steamwork** 的API获取。
    - `main.cpp` 为计算代码，用于计算游戏的性价比。
